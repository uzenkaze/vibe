1. 기본 사용법

상단 연도와 월을 선택하여 해당 월을 관리합니다. 데이터가 없는 달은 '신규 입력 모드'로 시작됩니다.

2. 이전 데이터 이월

[이전 달 데이터 이월] 버튼을 누르면 가장 최근에 입력했던 달의 항목들을 그대로 가져옵니다. 매달 반복되는 항목을 일일이 입력할 필요가 없습니다.

3. 지출 항목 관리

모든 항목은 분류, 내용, 금액, 비고 순으로 입력합니다. 금액은 숫자만 입력하며 자동으로 콤마(,) 처리가 됩니다.

4. 주요 계산 로직

  • 총 지출: 변동 지출 + 고정 지출 + 할부 원금 합계
  • 가용 잔액: 총 수입 - 총 지출
  • 순자산: (현금+비현금+부동산) - (부채+할부 잔액)

1. GitHub 연동의 장점

GitHub 연동을 통해 데이터를 클라우드에 안전하게 보관하고, 여러 기기(PC, 모바일 등)에서 동일한 데이터를 동기화하여 사용할 수 있습니다. 브라우저의 로컬 스토리지가 삭제되어도 데이터가 안전하게 유지됩니다.

2. 연동 설정 방법

오른쪽 상단의 [☁️ GitHub 연동] 버튼을 클릭하여 설정을 시작합니다.

  1. Personal Access Token: GitHub 계정에서 발급받은 토큰(fine-grained 또는 classic)을 입력합니다. (최소 repo 권한 필요)
  2. Repository: 데이터를 저장할 클라우드 저장소 이름(ex: username/repo-name)을 입력합니다.
  3. Path: 데이터 파일이 저장될 경로(ex: data/)를 지정합니다.

3. 자동 데이터 동기화

[자동 동기화] 옵션을 활성화하면 데이터를 저장할 때마다 자동으로 GitHub에 업로드(Push)되고, 앱을 새로고침하거나 월을 변경할 때 최신 데이터를 다운로드(Pull)하여 동기화합니다.

4. 수동 동기화 및 관리

설정 메뉴에서 필요할 때마다 업로드 또는 다운로드 버튼을 눌러 수동으로 데이터를 주고받을 수 있습니다. 동기화 성공 여부는 하단에 메시지로 표시됩니다.
