📖 자산 관리 앱 사용 매뉴얼

1. 화면 구성

앱은 크게 상단 헤더와 자산/수입/지출 영역으로 구성됩니다.

  • 연도 / 월 선택: 상단에서 조회할 연도와 월을 선택합니다. 월을 변경하면 해당 월의 데이터가 자동으로 불러와집니다.
  • 요약 카드: 총 수입 · 총 지출 · 가용 잔액 · 순자산이 한눈에 표시됩니다. 카드 위에 마우스를 올리면 계산 기준 툴팁이 표시됩니다.
  • 요약 보기 버튼: 각 섹션 상단의 요약 버튼을 누르면 해당 항목의 상세 요약 레이어가 팝업으로 표시됩니다.

2. 상단 헤더 버튼 안내

상단 오른쪽에는 다음 텍스트 메뉴들이 있으며, 마우스를 올리면 한글 툴팁이 나타납니다.

  • Card (카드할부): 카드 할부 관리 창을 엽니다. 이번 달 지출할 원금 및 수수료 합계가 총 지출에 반영됩니다.
  • Copy (데이터 복사): 가장 최근 월의 항목들을 현재 월로 이월합니다. 반복 지출 항목을 매달 새로 입력할 필요가 없습니다.
  • Cloud (클라우드 동기화): GitHub 연동 설정 창을 엽니다.
  • Data (데이터 관리): 데이터 백업(JSON 다운로드), 복원(JSON 붙여넣기), 월 데이터 삭제 기능을 제공합니다.
  • Manual (매뉴얼): 현재 보고 계신 이 화면입니다.
  • User (사용자 정보): 계정 정보를 확인하거나 수정합니다.
  • Logout (로그아웃): 현재 로그인된 상태를 해제합니다.

3. 자산·수입·지출 항목 입력

각 섹션에서 [+ 추가] 버튼을 누르면 새 행이 추가됩니다. 항목 입력 후 [저장]을 클릭하거나, 상단 Pension & Sync 바의 [💾 저장 & 동기화]로 데이터를 보존합니다.

  • 금액 칸에는 숫자만 입력하면 자동으로 콤마(,) 형식으로 변환됩니다.
  • 내용이 긴 항목은 툴팁으로 표시될 수 있습니다.
  • 변동/고정지출의 [상세] 버튼을 누르면 세부 항목을 분리하여 입력할 수 있습니다. (예: 식비 = 마트 + 배달 + 외식)
  • 모바일에서도 [상세] 버튼을 탭(터치)하면 세부 입력 창이 열립니다.

4. 연금 정보 및 저장 바

화면 상단의 흰색 막대(Pension & Sync Bar)에서 연금 및 저장 관련 기능을 수행합니다.

  • 📊 Pension Info: 예상 국민연금 정보를 입력할 수 있습니다. (세전/세후 예상 수령액, 수령 시기, 가입 기간 등 입력 시 자동 계산)
  • 💾 저장 & 동기화: 현재 화면의 데이터를 로컬 스토리지와 GitHub(연동 시)에 즉시 저장하고 동기화합니다.

이 앱의 핵심 목적

매달 수입을 기준으로 지출 금액을 빼면 얼마가 남는지(또는 부족한지)를 한눈에 파악하는 것입니다.
현금성 자산이나 부채는 현재 보유 현황만 볼 뿐, 월별 잔액 계산에는 포함되지 않습니다.

1. 할부 관리 열기

상단 헤더의 [💳 Card / 할부 관리] 버튼을 클릭하여 카드 할부 관리 창을 엽니다.

2. 할부 항목 입력

  • 사용일: 카드 사용 날짜를 선택합니다.
  • 카드사: 국민 / 신한 / 롯데 중 선택하거나 직접 입력합니다.
  • 내용: 할부 항목명 (예: 노트북, 냉장고 등)
  • 금액: 총 할부 금액을 입력합니다.
  • 회차: 현재 납부 회차 / 전체 회차 (예: 3 / 12 → 12개월 중 3회 납부 완료)
  • 월 원금: 총 금액 ÷ 전체 회차로 계산된 값이 자동으로 표시됩니다.
  • 잔액: (전체회차 - 현재회차) × 월 원금으로 자동 계산됩니다.
  • 종료일: 사용일 기준 전체 회차 후의 날짜가 자동으로 계산됩니다. (YY.MM 형식)

3. 행 색상 의미

  • 🟡 노란색 배경: 다음 달에 종료되는 할부 항목 (곧 완납 예정)
  • 🩶 회색 배경 + 흐릿하게: 종료일이 현재 조회 월보다 이전인 만료 완납된 할부
  • ⬜ 기본 흰색: 정상 진행 중인 할부

4. 정렬 및 합계

상단 정렬 드롭다운으로 사용일/금액/종료일 기준 오름/내림차순 정렬을 할 수 있습니다.
하단에는 전체 금액 합계 / 이번달 원금 합계(진행 중) / 할부 총 잔액이 자동으로 계산됩니다.

💡 이번달 원금 합계는 진행 중인 할부만 계산하며, 이 금액이 메인 화면 총 지출에 반영됩니다.

1. GitHub 연동의 장점

GitHub 연동을 통해 데이터를 클라우드에 안전하게 보관하고, 여러 기기(PC, 모바일 등)에서 동일한 데이터를 동기화하여 사용할 수 있습니다. 브라우저의 로컬 스토리지가 삭제되어도 데이터가 안전하게 유지됩니다.

2. 연동 설정 방법

오른쪽 상단의 [☁️ / GitHub] 버튼을 클릭하여 설정을 시작합니다.

  1. Personal Access Token (PAT): GitHub → Settings → Developer settings → Personal access tokens 에서 발급합니다. (최소 repo 권한 필요)
  2. 저장소 (Repository): 데이터를 저장할 저장소 이름 (예: username/repo-name)
  3. 브랜치: 기본값은 main. 저장소 기본 브랜치에 맞게 수정합니다.
  4. [💾 설정 저장] 버튼을 누르면 즉시 GitHub에서 기존 데이터를 가져오려고 시도합니다.

3. 자동 저장

[데이터 변경 시 자동 업로드] 옵션을 활성화하면 데이터를 저장할 때마다 자동으로 GitHub에 업로드(Push)됩니다.

4. 실시간 수동 저장

하단 연금 바의 [💾 실시간 저장] 버튼을 누르면 현재 데이터를 로컬 스토리지와 GitHub에 즉시 저장합니다.
저장 중에는 버튼이 ⏳로 바뀌고, 완료되면 ✅ 저장 완료로 잠시 표시됩니다.

5. 데이터 조회 및 복원

GitHub 연동 모달 내 📂 GitHub 데이터 관리 섹션에서 저장된 JSON 파일 목록을 불러오고, 선택한 파일의 내용을 미리 볼 수 있습니다.

새 기기에서 시작하려면 로그인 화면의 [☁️ GitHub에서 가져오기]를 이용하거나, JSON 파일을 화면에 드래그 앤 드롭하여 복원할 수 있습니다.
